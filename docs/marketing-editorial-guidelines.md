# Marketing Editorial Guidelines

Use this guide for user-facing copy in README, docs, templates, and site pages.

## Voice Rules

1. Lead with practical outcomes.
2. Keep sentences short and concrete.
3. State limits and prerequisites clearly.
4. Keep claims tied to verifiable repository assets.

## Style Rules

1. Use plain language and active voice.
2. Prefer checklists, steps, and tables over long paragraphs.
3. Avoid repeated adjectives and filler wording.
4. Keep product-line naming consistent.

## Phrase Policy

Avoid these words in user-facing copy:

- revolutionary
- cutting-edge
- seamless
- game-changing
- leverage
- unlock

## Product-Line Consistency

Always preserve this progression:

1. Sovra: open-source baseline
2. Klienta: paid white-label baseline
3. Clynova: paid healthcare baseline

## Proof-First Checklist

Back key claims with links:

- onboarding path -> `docs/customer-onboarding-dx.md` and `templates/onboarding/`
- production readiness -> `docs/production-readiness.md`
- security posture -> `SECURITY.md`
- discoverability layer -> `docs/seo-aeo-geo-guidelines.md` and `llms.txt`

## Pre-Publish Checks

1. Verify all links resolve.
2. Verify command snippets still run.
3. Validate copy against this style guide.
4. Reconfirm product-line framing remains accurate.
