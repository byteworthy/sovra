# Phase 6 — UI Design Contract

> Visual and interaction contract for billing management, admin dashboard, API key management, and monitoring integration.
> Extends Phase 3/5 design system without modification. All prior tokens carry forward unchanged.
> Target aesthetic: Linear settings pages / Vercel dashboard billing / Stripe admin console. Premium dark-mode-first.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Manual shadcn-compatible (no components.json -- extends Phase 2/3/5 hand-rolled system) |
| Preset | not applicable |
| Component library | Radix UI (already installed: dialog, dropdown-menu, select, slot, tabs, toast, slider) |
| Animation library | Framer Motion (already installed) |
| Icon library | lucide-react (already installed: ^0.469.0) |
| Font | Geist Sans (established in Phase 2 globals.css) |
| Font fallback | system-ui, -apple-system, sans-serif |

**Design philosophy:** Phase 6 introduces two distinct layout contexts that do not exist in prior phases.

First: tenant-scoped settings pages (billing, API keys) live inside the existing tenant dashboard shell (sidebar + main header). They follow the established page layout convention from Phases 3 and 5 with a single-column content well and tabbed sections where needed.

Second: the admin dashboard is a separate, distinct layout -- different from the tenant shell. Super-admin users reach it at `/admin` (no tenant slug). The admin layout uses a simplified sidebar with platform-scope navigation (Tenants, Users, Analytics, Audit Logs) and a visually distinct header that signals "you are operating outside tenant scope." Use a subtle amber accent on the admin nav to differentiate it from tenant-scoped blue. This prevents confusion between tenant views and platform views.

Deployment config pages (DEPL-01 through DEPL-05) are documentation-first and have no custom UI -- they are rendered markdown or static config files. No component contract needed for deployment beyond the standard page shell.

Monitoring integrations (Sentry, PostHog, health checks, custom metrics) are backend-only except for a status readout in the admin dashboard. No dedicated monitoring pages in the frontend for Phase 6.

**Source of truth:** `packages/web/app/globals.css` CSS variables. `packages/web/lib/motion.ts` animation presets. Do not redefine either. Reference only.

### 60/30/10 Distribution (Phase 6 scope)

- 60% dominant: `--background` (#09090B) -- billing page, API keys page, admin dashboard background, all settings page backgrounds
- 30% secondary: `--card` / `--surface-elevated` -- plan cards, usage summary cards, API key rows, admin tenant rows, audit log rows, admin stat cards
- 10% accent: `--primary` (#3B82F6) -- reserved exclusively for:
  1. Primary CTA buttons ("Create API key", "Upgrade plan", "Save", "Export")
  2. Active sidebar nav item -- inherited Phase 2 pattern (tenant-scoped pages)
  3. Focus ring on keyboard-focused interactive elements (2px ring)
  4. Usage bar fill below 70% threshold
  5. Current plan card subtle background tint
  6. Pro plan tier badge (note: different shade per token above)

**Admin scope exception:** Admin layout active nav item uses amber (`#FBBF24`) instead of blue. This is the only context where a non-blue accent is used for nav activation. All CTA buttons in admin pages still use `--primary` blue.

## Animation Contracts

### Inherited from Phase 3/5 (reference -- do not redefine)

All VARIANTS and TRANSITIONS from `packages/web/lib/motion.ts` are available. Specifically re-used in Phase 6:
- `cardEnter` -- create API key dialog
- `listItem` -- admin tenant rows, API key rows on initial load (stagger 0.03s)

### Phase 6 animation additions

No new animation variants required. All interactions map to existing presets.

Add these CSS utility classes to `globals.css` (not motion.ts -- they are CSS-only):

```css
/* API key reveal -- highlight copy target */
.key-reveal-highlight {
  animation: keyReveal 0.4s ease-out;
}
@keyframes keyReveal {
  0%  { background-color: rgba(59,130,246,0.2); }
  100%{ background-color: transparent; }
}
```

| Component | Animation | Timing |
|-----------|-----------|--------|
