## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Lemon Squeezy webhook -> /api/billing/webhook | Untrusted HTTP POST from external service; must verify HMAC signature |
| Client -> /api/billing/portal | Authenticated user requesting portal redirect; must verify tenant membership |
| External -> /api/health | Public endpoint; must not leak internal state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | /api/billing/webhook | mitigate | HMAC-SHA256 signature verification with timingSafeEqual; reject if X-Signature header missing or invalid |
| T-06-02 | Tampering | subscription state | mitigate | Subscription status only updated via verified webhook handler; no client-side subscription mutation endpoints |
| T-06-03 | Information Disclosure | /api/health | accept | Health endpoint returns only { status: 'ok' }; no DB connection details or version info |
| T-06-04 | Spoofing | webhook replay | accept | Low risk: LS signs each payload uniquely; add timestamp freshness check in v2 if needed |
</threat_model>
