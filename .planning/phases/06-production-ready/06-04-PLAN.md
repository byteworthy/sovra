## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> billing/portal | Authenticated tenant user accessing billing; must verify tenant membership |
| Client -> API key create | Raw key exposed once in browser; user responsible for storing securely |
| Client -> API key revoke | Must verify key belongs to user's tenant |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-14 | Information Disclosure | CreateApiKeyDialog | mitigate | Raw key shown once, dialog backdrop disabled during reveal, copy-then-dismiss pattern forces user acknowledgment |
| T-06-15 | Tampering | API key permissions UI | mitigate | Permissions validated server-side in POST /api/keys; UI checkboxes are convenience only |
| T-06-16 | Denial of Service | billing page data load | accept | Server-side data fetch with error boundary; error toast per UI spec copy |
</threat_model>
