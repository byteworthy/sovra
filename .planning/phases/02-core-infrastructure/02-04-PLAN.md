| Client-side auth state -> UI rendering | Auth guard determines what user sees |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Spoofing | Login form | mitigate | Zod validates email format and password presence; Supabase handles rate limiting and brute-force protection; generic error messages do not reveal whether email exists |
| T-02-13 | Information Disclosure | Error display from searchParams | mitigate | Error messages displayed via Alert component use React text nodes not raw HTML insertion; searchParams.error rendered as escaped text content |
| T-02-14 | Cross-Site Scripting | OAuth error in URL | mitigate | Error param from searchParams rendered as React text child (auto-escaped by React); encodeURIComponent used in callback route (Plan 01 Task 3) |
| T-02-15 | Spoofing | Auth guard | mitigate | Auth guard calls getUser() which validates JWT server-side via Supabase; skeleton shown during validation prevents flash of protected content |
| T-02-16 | Information Disclosure | Forgot password | mitigate | Success shown regardless of whether email exists -- prevents email enumeration; 60s cooldown prevents abuse |
