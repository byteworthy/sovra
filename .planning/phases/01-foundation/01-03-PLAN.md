   Users table - user can read/update own profile:
   ```sql
   create policy "users_select_own" on users
     for select to authenticated
     using (id = (select auth.uid()));

   create policy "users_insert_own" on users
     for insert to authenticated
     with check (id = (select auth.uid()));

   create policy "users_update_own" on users
     for update to authenticated
     using (id = (select auth.uid()));
   ```

   Tenants table - members can view their tenant:
   ```sql
   create policy "tenants_select_member" on tenants
     for select to authenticated
     using (id = (select get_current_tenant_id()));

   create policy "tenants_insert_authenticated" on tenants
     for insert to authenticated
     with check (true);

   create policy "tenants_update_owner" on tenants
     for update to authenticated
     using (id = (select get_current_tenant_id()));
   ```

   Tenant_users - members can see other members in their tenant:
   ```sql
   create policy "tenant_users_select" on tenant_users
     for select to authenticated
     using (tenant_id = (select get_current_tenant_id()));

   create policy "tenant_users_insert" on tenant_users
     for insert to authenticated
     with check (tenant_id = (select get_current_tenant_id()));

   create policy "tenant_users_delete" on tenant_users
     for delete to authenticated
     using (tenant_id = (select get_current_tenant_id()));
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RLS policy boundary | Tenant A's JWT must never access Tenant B's data |
| Service role | Bypasses all RLS - must only be used in migrations and admin scripts |
| auth.uid() / auth.jwt() | Supabase auth functions trusted within RLS policies |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-01-08 | I (Information Disclosure) | RLS policies | mitigate | Every tenant-scoped table has RLS enabled. All policies use `(select get_current_tenant_id())` with query-level caching. Audit logs are append-only (no UPDATE/DELETE policies). |
| T-01-09 | E (Elevation of Privilege) | Service role key | mitigate | Service role key documented in .env.example as server-only. Never exposed via NEXT_PUBLIC_ prefix. App code uses anon key + user JWT only. |
| T-01-10 | T (Tampering) | get_current_tenant_id() | mitigate | Function is `security definer` - runs with definer privileges, not caller. Looks up tenant_users table which is also RLS-protected. Users cannot forge tenant_id. |
| T-01-11 | I (Information Disclosure) | Feature flags global read | accept | Global feature flags (null tenant_id) are intentionally readable by all authenticated users. They contain feature gate names, not sensitive data. |
| T-01-12 | S (Spoofing) | auth.uid() in RLS | mitigate | auth.uid() is extracted from the verified JWT by Supabase - not user-controllable. Token verification happens in the PostgREST layer before the query runs. |
</threat_model>
