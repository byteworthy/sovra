1. Create `docker/compose.dev.yaml`:
   ```yaml
   name: sovra-dev

   services:
     web:
       build:
         context: ../packages/web
         dockerfile: Dockerfile.dev
       ports:
         - "3000:3000"
       volumes:
         - ../packages/web:/app
         - ../packages/shared:/shared
         - /app/node_modules
         - /app/.next
       environment:
         - NODE_ENV=development
         - NEXT_PUBLIC_SUPABASE_URL=http://localhost:54321
         - NEXT_PUBLIC_SUPABASE_ANON_KEY=${SUPABASE_ANON_KEY:-your-anon-key}
       network_mode: host

     worker:
       build:
         context: ../packages/worker
         dockerfile: Dockerfile.dev
       ports:
         - "8080:8080"
         - "50051:50051"
       volumes:
         - ../packages/worker:/app
         - go-cache:/root/.cache/go
       environment:
         - DATABASE_URL=postgresql://postgres:postgres@localhost:54322/postgres
         - HTTP_PORT=8080
         - GRPC_PORT=50051
         - GO_ENV=development
       network_mode: host

   volumes:
     go-cache:
   ```

   NOTE on `network_mode: host`: This allows containers to connect to Supabase running on the host (via `supabase start`). The `ports` directive is documentation-only when using host networking on Linux. On macOS Docker Desktop, host networking behaves differently - the containers will still need `host.docker.internal`. However, `network_mode: host` is the standard pattern from the research. If issues arise on macOS, switch to a bridge network with `extra_hosts: ["host.docker.internal:host-gateway"]`.

2. Create `docker/compose.prod.yaml`:

   ```yaml
         dockerfile: Dockerfile
       ports:

   ```

3. Create `packages/web/Dockerfile` (production multi-stage):
   ```dockerfile
   # Dependencies
   FROM node:20-alpine AS deps
   RUN corepack enable && corepack prepare pnpm@9.0.0 --activate
   WORKDIR /app
   COPY package.json pnpm-lock.yaml* ./
   RUN pnpm install --frozen-lockfile --prod=false

   # Build
   FROM node:20-alpine AS builder
   RUN corepack enable && corepack prepare pnpm@9.0.0 --activate
   WORKDIR /app
   COPY --from=deps /app/node_modules ./node_modules
   COPY . .

   ENV NEXT_TELEMETRY_DISABLED=1
   ENV NODE_ENV=production

   RUN pnpm build

   # Runtime
   FROM node:20-alpine AS runner
   WORKDIR /app

   ENV NODE_ENV=production
   ENV NEXT_TELEMETRY_DISABLED=1

   RUN addgroup --system --gid 1001 nodejs
   RUN adduser --system --uid 1001 nextjs

   COPY --from=builder /app/public ./public
   COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
   COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

   USER nextjs

   EXPOSE 3000

   ENV PORT=3000
   ENV HOSTNAME="0.0.0.0"

   CMD ["node", "server.js"]
   ```

   NOTE: This requires `output: 'standalone'` in next.config.ts. Check the existing next.config.ts - if it doesn't have standalone output, add it:
   ```typescript
   import type { NextConfig } from 'next'
   const nextConfig: NextConfig = {
     output: 'standalone',
   }
   export default nextConfig
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker image layers | Build artifacts may contain secrets if not careful |
| Docker socket | Container escape risk if mounted |
| Environment variables in compose | Secrets passed at runtime, not baked into images |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-01-13 | I (Information Disclosure) | Dockerfile layers | mitigate | Multi-stage build - only final artifacts copied to runtime stage. No source code, no node_modules dev deps, no .env files in production image. .dockerignore excludes .env* files. |
| T-01-14 | E (Elevation of Privilege) | Docker containers | mitigate | Production images run as non-root user (nextjs uid 1001, worker uid 1000). No Docker socket mounted in any container. No privileged mode. |
| T-01-15 | I (Information Disclosure) | compose.prod.yaml env vars | mitigate | All secrets passed via environment variables at runtime (${VAR} syntax), never hardcoded in compose file. SUPABASE_SERVICE_ROLE_KEY never uses NEXT_PUBLIC_ prefix. |
| T-01-16 | D (Denial of Service) | Health checks | accept | Health endpoints are unauthenticated (intentional - required for orchestrator/LB). They return minimal data only. Rate limiting not needed for health checks at Phase 1 scale. |
</threat_model>
