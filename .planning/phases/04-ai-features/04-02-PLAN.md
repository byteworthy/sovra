           await supabase.from('agents').update({ status: 'idle' }).eq('id', agentId)
           // Track tool executions
           const TOOL_COSTS: Record<string, number> = {
             web_search: 0.001, web_fetch: 0.0001,
             file_read: 0, file_write: 0, file_list: 0,

               await supabase.from('tool_executions').insert({

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Next.js -> Go worker | MCP client connects to worker over Docker internal network |
| User -> chat API | Chat messages from authenticated users trigger tool execution |
| Tool results -> LLM | Tool output fed back to model as tool_result messages |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Denial of Service | chat route | mitigate | AbortSignal.timeout(30000) on every callTool; maxSteps=10 caps total tool invocations |
| T-04-09 | Elevation of Privilege | tool_executions | mitigate | tenant_id extracted from agent record (RLS-fetched), not from user input |
| T-04-10 | Information Disclosure | MCP connection | mitigate | WORKER_MCP_URL is internal Docker URL, not exposed; graceful degradation on failure |
| T-04-11 | Tampering | tool cost tracking | accept | Cost estimates are static constants per tool type; actual billing not derived from these values |
| T-04-12 | Spoofing | chat route | mitigate | Auth check (getUser) already exists before any tool execution path |
</threat_model>
