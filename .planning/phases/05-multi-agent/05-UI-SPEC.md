# Phase 5 — UI Design Contract

> Visual and interaction contract for workspace management, multi-agent collaboration, memory strategy configuration, and real-time agent status.
> Extends Phase 3 design system without modification. All Phase 3 tokens carry forward unchanged.
> Target aesthetic: Linear project board / Vercel team workspace / Cursor multi-pane. Premium dark-mode-first.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Manual shadcn-compatible (no components.json — extends Phase 2/3 hand-rolled system) |
| Preset | not applicable |
| Component library | Radix UI (already installed: dialog, dropdown-menu, select, slot, tabs, toast, slider) |
| Animation library | Framer Motion (already installed) |
| Icon library | lucide-react (already installed: ^0.469.0) |
| Font | Geist Sans (established in Phase 2 globals.css) |
| Font fallback | system-ui, -apple-system, sans-serif |

**Design philosophy:** Phase 5 adds a new top-level nav section — Workspaces — sitting alongside Agents in the tenant sidebar. The workspace detail view is the centerpiece: a live collaboration canvas showing agents operating in real-time, shared memory state, and agent-to-agent message traffic. The real-time layer (Socket.IO) should feel invisible — status updates animate smoothly, never jarring. Memory strategy configuration is surfaced as a workspace-level settings sheet, not a separate page.

**Source of truth:** `packages/web/app/globals.css` CSS variables. `packages/web/lib/motion.ts` animation presets. Do not redefine either. Reference only.
