# Phase 3 — UI Design Contract

> Visual and interaction contract for agent management CRUD and real-time chat interface.
> Extends the Phase 2 design system without modification. All Phase 2 tokens carry forward unchanged.
> Target aesthetic: Linear agent view / Cursor IDE chat / Vercel AI Playground.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Manual shadcn-compatible (no components.json — extends Phase 2 hand-rolled system) |
| Preset | not applicable |
| Component library | Radix UI (already installed: dialog, dropdown-menu, select, slot, tabs, toast, slider) |
| Animation library | Framer Motion (already installed from Phase 2) |
| Icon library | lucide-react (already installed: ^0.469.0) |
| Font | Geist Sans (established in Phase 2 globals.css) |
| Font fallback | system-ui, -apple-system, sans-serif |

**Design philosophy:** Carry forward the Phase 2 dark-first glass morphism system. Agent pages use the tenant dashboard shell (sidebar + main header + content area). Chat interface uses a two-column split within the main content area: conversation list on the left, active chat on the right. Streaming tokens render as they arrive with a subtle typing cursor. Agent configuration panels use a tabbed drawer/sheet pattern — never navigate away from chat to configure the agent.

**Source of truth:** `packages/web/app/globals.css` CSS variables. `packages/web/lib/motion.ts` animation presets. Do not redefine either. Reference only.

## Animation Contracts

### Phase 2 presets (reference — do not redefine)

All TRANSITIONS and VARIANTS from `packages/web/lib/motion.ts` are available. Reference them, do not duplicate.

### Phase 3 animation additions

Add these entries to `packages/web/lib/motion.ts`:

```typescript
// Add to VARIANTS object in lib/motion.ts

// Chat message entrance (user and assistant)
messageEnter: {
  initial: { opacity: 0, y: 8 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.18, ease: [0.32, 0.72, 0, 1] },
},

// Streaming token pulse (applied to cursor element)
// Use CSS keyframe, not Framer Motion - lower overhead during streaming
// .streaming-cursor { animation: blink 0.9s step-end infinite; }
// @keyframes blink { 0%,100%{opacity:1} 50%{opacity:0} }

// Agent status pulse (running state)
// Use CSS keyframe for continuous animation
// .agent-status-running { animation: statusPulse 2s ease-in-out infinite; }
// @keyframes statusPulse { 0%,100%{opacity:1;transform:scale(1)} 50%{opacity:0.6;transform:scale(0.85)} }

// Slide in from right (agent config sheet)
slideInRight: {
  initial: { x: '100%', opacity: 0 },
  animate: { x: 0, opacity: 1 },
  exit: { x: '100%', opacity: 0 },
  transition: TRANSITIONS.spring,  // reference existing
},

// Jump to latest pill
jumpToLatest: {
  initial: { opacity: 0, y: 8, scale: 0.95 },
  animate: { opacity: 1, y: 0, scale: 1 },
  exit: { opacity: 0, y: 8, scale: 0.95 },
  transition: TRANSITIONS.springFast,  // reference existing
},
```

### Per-Component Animation Specs

| Component | Animation | Timing |
|-----------|-----------|--------|
| Agent card grid enter | Stagger 0.04s, `listItem` variant | 200ms per card |
| Create agent dialog open | `cardEnter` with AnimatePresence | 350ms |
| Create agent dialog close | Reverse `cardEnter` (scale 1→0.98, opacity 1→0) | 200ms |
| Agent config sheet open | `slideInRight` — spring stiffness 400, damping 30 | ~250ms |
| Agent config sheet close | `slideInRight` exit | ~200ms |
| Tab switch in config sheet | Content fade 0→1, translateX ±8px | 150ms |
| Chat message appear | `messageEnter` for each message | 180ms |
| Streaming token appearance | No animation — raw text append (performance) | — |
| Streaming cursor blink | CSS keyframe, 0.9s step-end | continuous |
| Agent status running dot | CSS keyframe pulse 2s ease-in-out | continuous |
| Jump to latest pill appear | `jumpToLatest` with AnimatePresence | spring fast |
| New conversation click | Active item highlight `listItem` | 150ms |
| Tool call pill expand/collapse | Height auto with AnimatePresence | 200ms spring |
| Unsaved changes warning appear | Fade in opacity 0→1 | 150ms |
| Delete conversation confirm row | Same inline pattern as Phase 2 member row remove | 250ms |

## Interaction Choreography

### Create Agent Flow

```
1. User clicks "+ Create agent"
2. Dialog opens — cardEnter animation (350ms)
3. Name input autofocused
4. User fills name (required), optionally fills description, provider, model, system prompt
5. "Create agent" clicked:
   a. Button → loading state (Loader2 spinner, "Creating...")
   b. Validation: name required — if empty, input error state (border-destructive)
   c. Success:
      → Dialog closes (reverse cardEnter, 200ms)
      → New agent card appears in grid (listItem animation, stagger from top)
      → Toast: "Agent created"
   d. Error:
      → Inline error alert below form
      → Button returns to default state
6. Cancel / click backdrop → dialog closes, no save
```

### Open Agent Chat Flow

```
1. User clicks agent card
2. Router push to /t/{slug}/agents/{agentId}/chat
3. Chat area enters with pageEnter (Phase 2 preset, 200ms)
4. If prior conversations exist: most recent conversation selected, messages load with skeleton
5. If no conversations: empty state shown in chat area
6. Textarea autofocused
```

### Send Message Flow

```
1. User types in textarea
2. Send button enabled as soon as first character typed
3. Cmd/Ctrl+Enter OR click send:
   a. User message bubble appears immediately (messageEnter, 180ms)
   b. Textarea clears
   c. Send button → loading state (Loader2)
   d. Textarea disabled
   e. Assistant thinking indicator: 3-dot pulse animation appears in left-aligned bubble position
      (standard ellipsis animation: each dot opacity 0→1→0 staggered 0.2s)
   f. As tokens stream in: dots replaced by message text, cursor blinks at end
   g. On stream complete: cursor disappears, send button re-enables, textarea re-enables, autofocused
4. On error: error toast (bottom-right) + send button re-enables
   Error toast message per error type (see Copywriting)
5. Auto-scroll to bottom after each new message (unless user scrolled up)
```

### Agent Configuration Flow

```
1. User clicks Settings2 icon in chat header
2. Config sheet slides in from right (slideInRight, spring 400/30)
3. "General" tab active by default
4. User edits fields — "Unsaved changes" warning appears (fade in, 150ms) on first change
5. Tab switch: content fades out + translateX, new tab fades in + translateX
6. "Save changes" clicked:
   a. Button → loading
   b. Success: sheet closes, toast "Changes saved"
   c. Error: inline error in sheet footer, button returns to default
7. "Cancel" or X clicked:
   a. If dirty: dialog appears "Discard unsaved changes?" — "Discard" + "Keep editing"
   b. If clean: sheet closes immediately
```

### Delete Agent Flow

```
1. Agent card → MoreHorizontal → "Delete agent"
2. Dropdown closes
3. Confirmation dialog opens (cardEnter):
   "Delete this agent?" heading
   Body copy (see Copywriting)
   "Delete agent" (Button variant="destructive") + "Keep agent" (ghost)
4. "Delete agent" clicked:
   a. Button → loading
   b. Success:
      → Dialog closes
      → Agent card fades out and collapses (opacity 1→0, height 0→0, 300ms)
      → Toast "Agent deleted"
      → If user was in that agent's chat: redirect to /t/{slug}/agents
5. Error: inline error in dialog, button returns to default
```

### Delete Conversation Flow

```
1. ConversationListItem right-click OR hover → MoreHorizontal → "Delete conversation"
2. Confirmation inline below the item (same pattern as Phase 2 member row remove):
   "Delete this conversation? Your history will be permanently deleted."
   "Delete" (destructive, h-8 text-xs) + "Cancel" (ghost, h-8 text-xs)
3. "Delete" clicked:
   a. Item fades out (opacity 1→0, height 0, 250ms)
   b. If it was active conversation: adjacent conversation selected, or empty state shown
   c. Toast: "Conversation deleted"
```

## Accessibility Contract

**Inherits all Phase 2 accessibility requirements. Phase 3 additions:**

- Chat message feed: `role="log"` `aria-live="polite"` `aria-label="Conversation with {agent_name}"`. New messages announced to screen readers as they complete (not during stream).
- Streaming: `aria-busy="true"` on the feed container during active stream. `aria-label="Agent is responding"` on the thinking indicator.
- Chat input textarea: `aria-label="Message {agent_name}"`. `aria-disabled="true"` when agent is running.
- Send button: `aria-label="Send message"`. `aria-disabled="true"` when empty or agent running.
- Agent status badge: `aria-label="Status: {status}"`.
- Config sheet: `role="dialog"` `aria-labelledby` pointing to "Configure agent" heading. Focus trap active while open.
- Tab panel in config sheet: Radix Tabs — keyboard navigation (arrow keys) included.
- Temperature slider: `aria-label="Temperature"` `aria-valuenow={value}` `aria-valuemin={0}` `aria-valuemax={2}`. Radix Slider provides these automatically.
- Tool call pill: `aria-expanded={isExpanded}` `role="button"` with keyboard activation.
- Jump to latest pill: `role="button"` `aria-label="Jump to latest message"`.
- Code blocks: `role="region"` `aria-label="Code block"`. Copy button: `aria-label="Copy code"`.
- Agent card: `role="article"` or use `<article>` element. Keyboard: Enter/Space to navigate to chat.
- Conversation list items: `role="option"` inside `role="listbox"` for the conversation list. `aria-selected="true"` on active item.
- Empty state CTAs: descriptive `aria-label` — "Create your first agent" not just "Create".
- Focus management: on dialog/sheet open, first focusable element receives focus. On close, focus returns to trigger.
- Minimum touch targets: Send button h-9 w-9 (36px — acceptable for icon-only on desktop). All destructive actions h-8 minimum.
- Color contrast: streaming cursor (#3B82F6) on dark (#0D0D0F) = 4.9:1 ratio (passes AA). User bubble (#1E3A5F bg) with #FAFAFA text = 8.1:1 (passes AAA).
