| Boundary | Description |
|----------|-------------|
| Client form data -> Server Actions | User-controlled form values cross to server |
| URL params (agentId) -> DB queries | User can manipulate URL to access other agents |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | Agent form submission | mitigate | Zod schema validates all fields before DB insert; rejects invalid provider/temperature/tokens |
| T-03-07 | Elevation of Privilege | Agent delete from card menu | mitigate | deleteAgent server action checks agent:delete permission via hasPermission() |
| T-03-08 | Information Disclosure | Agent list page | mitigate | listAgents query filtered by tenant_id; RLS prevents cross-tenant reads |
</threat_model>
